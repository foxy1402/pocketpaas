Import & Sequential Deploy

Designed for ephemeral storage (e.g. free-tier PaaS with limited disk). Imports configs, then for each app in order: pulls the image → starts the app → immediately prunes the rootfs from disk before moving to the next app. This prevents OOM crashes caused by holding multiple image filesystems on disk at once.

About rootfs: Container image filesystems are not included in the export (too large). After importing on a fresh container, use Sequential Deploy above (ephemeral storage) or Pull All (persistent storage) to re-download images.
